# 第 1 周 MVP 范围说明

## 1. 文档目标

本文档用于确认“企业级 AI 知识库问答后端平台”在第 1 周的建设范围，确保项目先完成单体 Spring Boot MVP 的总体设计与骨架初始化，不提前展开复杂 AI 能力实现。

第 1 周的核心目标不是把系统全部做完，而是把后续开发必须依赖的边界、结构、表设计、接口清单和工程骨架先定下来。

## 2. 项目名称确认

- 项目名称：企业级 AI 知识库问答后端平台
- 项目定位：面向企业内部知识问答场景的单体 Spring Boot 后端 MVP
- 当前阶段目标：完成第 1 版方案收敛、工程初始化、数据库与接口设计

## 3. 第 1 周保留范围

第 1 周只保留以下内容：

- 项目名称确认
- MVP 边界确认
- 技术栈定稿
- 核心模块划分
- 核心数据库表设计
- 项目目录结构
- 第 1 批接口清单
- AI coding 工作流
- Spring Boot 项目骨架初始化

## 4. MVP 边界确认

### 4.1 第 1 版要解决的问题

第 1 版只关注一个最小业务闭环：

1. 用户可以注册、登录系统
2. 管理员或用户可以创建知识库
3. 用户可以上传文档到知识库
4. 系统可以记录文档处理任务状态
5. 用户可以发起基础问答
6. 系统可以保存会话与消息历史
7. 管理端可以查看基础数据和文档状态

### 4.2 第 1 版功能范围

第 1 版只保留这些基础能力：

- 用户注册
- 用户登录
- 获取当前登录用户信息
- 创建知识库
- 查询知识库列表与详情
- 上传文档
- 查询文档列表与详情
- 记录文档解析/切片任务
- 基础问答接口
- 会话创建与历史查询
- 问答消息记录
- 管理端基础接口

### 4.3 第 1 版先不深入的内容

以下内容不纳入第 1 周深入设计和实现：

- 向量库选型
- Embedding 实现
- LLM 接入细节
- 复杂权限体系
- Tool Calling
- 多知识库路由
- 复杂审核流

这些内容后续再进入专题设计，避免第 1 周范围失控。

## 5. 技术栈定稿

第 1 周固定采用以下技术栈：

- Java 17
- Spring Boot 3.x
- Spring Web
- Spring Validation
- Spring Security
- JWT
- MyBatis-Plus
- MySQL 8
- Redis
- Flyway
- Lombok
- springdoc-openapi
- Logback
- JUnit 5

## 6. 交付目标

第 1 周结束前，项目应达到以下状态：

- 已明确项目名称和 MVP 范围
- 已确定统一技术栈
- 已完成核心模块划分
- 已完成核心数据库表设计
- 已输出第 1 批接口清单
- 已确定项目目录结构
- 已形成 AI coding 工作流规范
- 已完成 Spring Boot 工程骨架初始化

## 7. 第 1 周不追求的结果

第 1 周不追求以下结果：

- 完整 RAG 问答链路跑通
- 文档解析、切片、检索全部实现
- 向量检索方案最终落地
- 模型服务正式接入
- 管理后台页面开发完成
- 高可用、分布式、微服务化设计

## 8. 第 1 周产出清单

第 1 周建议产出以下内容：

- `docs/mvp-scope.md`
- `docs/backend-modules.md`
- `docs/database-design.md`
- `docs/api-list-week1.md`
- `README.md`
- `pom.xml`
- `src/main/java` 基础包结构
- `src/main/resources/application.yml`
- `src/main/resources/db/migration/V1__init.sql`

## 9. 结论

第 1 周只做“定边界、定结构、定表、定接口、起骨架”。

这样做的目的，是让项目尽快进入“可以稳定继续开发”的状态，而不是过早陷入向量库、模型接入和复杂权限等高不确定性问题。
